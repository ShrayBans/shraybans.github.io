React Native

Reusing code for mobile applications

Posted by Shray Bansal on July 28, 2016

For decades, making a mobile application has been the hardest thing. You had to learn various different languages to learn development for iOS and a different language for Android. In addition, web developers haven’t been able to even touch the realm of mobile apps because the syntax and the languages are so different. However, the stars have aligned and Javascript has brought together the communities with a common framework: React. Facebook has developed something magical.

Introducing React Native: a platform that allows for mobile development for both iOS and Android

My Journey into React Native

At first I thought that I would be able to port my beloved React applications that I poured my blood, sweat and tears into directly into a mobile app. With a couple of clicks, I would have a fully functional mobile app. However, I quickly discovered that React Native had an entirely different syntax and everything looked like a foreign language. This is what a sample React Native app looks like:

My ambition for building the mobile app quickly disappeared, but I had a couple of hours lying around, so I decided to give it a try. I wondered: what are these “TouchableHighlights” and these “Views”? With a quick Google Search, I was able to understand that these were just another way of naming buttons and divs. Soon after I understood the basic syntax, I was dove into tougher subjects such as page navigation. I read through a couple tutorials that were littered with foreign-looking code:

Looking at it, it looks just like routes linking different pages together (similar to react router). However, notice that there are also props being passed into each route.

Summary: React Native can provide developers with the best of multiple worlds:

  • Flexibility of using a single language to port to both Android and iOS devices
  • Programming power and design concepts typically used in React
